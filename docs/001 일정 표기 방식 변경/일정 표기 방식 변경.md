- [x] 일정 및 작업 API가 v2.1로 변경되었다.
- [x] 일정/및 작업은 이제 offset을 백엔드에 보내는 대신, IANA에 맞춰 zoneId를 보낸다.

# 고려해야 하는 것
- [x] 일정 및 작업 API 호출 시 zoneId를 포함해야 한다.
- [x] 일정 및 작업 표기 시, 사용자의 로컬 타임존을 고려해야 한다.
- [x] 현재 이 앱은 서울에서 배포되는 것을 가정하므로, 시간대를 Asia/Seoul로 설정해야 한다.
- [x] 일정 및 작업 생성/수정 시, zoneId를 포함하여 백엔드와의 시간대 불일치를 방지해야 한다.
- [x] 일정 및 작업 생성/수정 시, 사용자가 직접 입력한 시간 정보를 올바르게 처리해야 하므로, toISOString과 같은 UTC로 강제 변환되는 함수를 사용하지 않아야 한다. 사용자의 벽시계 시간대 사용을 유지해야 한다.
- [x] 일정 및 작업 목록, 상세 화면에서 사용자의 시간대 정보를 API 호출하여 받아온 뒤 일치시키는 로직이 존재한다. 이를 필요하다면 제거하고, 정확한 시간대 처리를 위해 zoneId를 사용하도록 수정해야 한다.
